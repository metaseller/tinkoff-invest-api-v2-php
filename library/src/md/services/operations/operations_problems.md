# Особенности методов сервиса операций

<p>При работе с методами получения списка операций учитывайте:</p>
<ol>
<li><p>ID операций могут меняться со временем.</p>
<p>Не стоит ориентироваться на ID операций как на первичный ключ. Если вам нужно перезагрузить историю операций, удалите старые ID операции при повторном запросе операции.</p>
</li>
<li><p>В ряде случаев <b>trade_id</b> в операциях может отличаться от <b>trade_id</b> сервиса заявок.</p>
<p>Для сбора информации по совершенным сделкам, включая идентификатор, рекомендуется следующий подход: первым шагом из операций получаются <b>id</b>, а затем методом <a href="/invest/services/orders/methods#getorderstat">GetOrderState</a> достаются идентификаторы сделок, используя <b>id</b> операции в качестве параметра <b>order_id</b>.</p>
</li>
<li><p>По инструментам, прошедшим корпоративные действия, история операций может быть не полной — как в API, так и в мобильном приложении. </p>
<p>Чтобы получить точную информацию, используйте <a href="/invest/services/operations/methods#getbrokerreport">брокерский отчет</a>.</p>
</li>
<li><p>В ID комиссий приходит <b>parent_operation_id</b>.</p>
</li>
<li><p>В методе <a href="/invest/services/operations/methods#getoperations">getOperations</a> информация, которая возвращается по операциям с опционами, отображается некорректно. </p>
<p>Чтобы получить список операций с опционами, используйте метод <a href="/invest/services/operations/methods#getoperationsbycursor">getOperationsByCursor</a>.</p>
</li>
<li><p>Если вы получаете список операций через метод <a href="/invest/services/operations/methods#getoperationsbycursor">getOperationsByCursor</a>, используйте limit &gt; 2 — это поможет избежать проблем
с задублированными операциями и невозможностью перемещаться по cursor. </p>
</li>
<li><p>Все типы операций, подразумевающие начисление и выплаты по погашению облигаций, вариационной марже, дивидендам и налогам не отображают данные о количестве инструментов, по которым произошли начисления.</p>
</li>
</ol>

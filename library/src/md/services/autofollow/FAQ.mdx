# FAQ

Основная информация для ведущих размещена на странице [Всё об автоследовании для авторов стратегий](https://www.tbank.ru/invest/help/brokerage/account/strategy-authors).
Вопросы, связанные с сервисом, размещены ниже.

## Где взять данные для авторизации?

Для авторизации в сервисе используется токен T-Invest API.
[Как выпустить и отозвать токен](/invest/intro/intro/token).

В сервисе автоследования нужно использовать токен с `full-access` набором прав.

## Как получить идентификатор стратегии `strategy_id`?

Список стратегий ведущего можно получить через метод [getAutofollowStrategies](/invest/api/autofollow/get-autofollow-instruments). В ответе вернется массив стратегий с идентификаторами и текущий статус.
Метод вызывается без дополнительных параметров.

## Какие инструменты доступны для автоследования?

Массив доступных инструментов и их идентификаторов возвращается в методе 
[getAutofollowInstruments](/invest/api/autofollow/get-autofollow-instruments).

## Как создать новый сигнал?

Для создания сигнала используйте метод [createAutofollowSignal](/invest/api/autofollow/create-autofollow-signal).
В параметрах метода передайте:
- идентификатор стратегии — **strategy_id**;
- идентификатор инструмента — **instrument_id**;
- направление **direction** — `buy` или `sell`;
- количество лотов — **lots**, может быть дробным.

## Как часто можно создавать сигнал?

Сейчас у [метода](/invest/api/autofollow/create-autofollow-signal) есть ограничение: создавать сигнал 
можно не чаще, чем раз в 15 секунд.

## Как создать отложенный сигнал?

Для создания отложенного сигнала используйте метод [createAutofollowStopSignal](/invest/api/autofollow/create-autofollow-signal).
В параметрах метода передайте:
- идентификатор стратегии — **strategy_id**;
- идентификатор инструмента — **instrument_id**;
- тип отложенной заявки — **stop_order_type**: `take_profit` или `stop_loss`;
- направление **direction** — `buy` или `sell`;
- количество лотов — **lots**, может быть дробным;
- цену активации — **stop_price**;
- опционально — дату экспирации: **expire_date**.

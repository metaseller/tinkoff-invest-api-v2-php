# REST
Для работы c использованием REST можно использовать [API](/invest/api) или [Swagger T-Invest API](https://russianinvestments.github.io/investAPI/swagger-ui/).
C помощью него вы можете тестировать методы T-Invest API.


## API

Встроенный в документацию [инструмент](/invest/api) — удобный способ для тестирования методов T-Invest API, отладки и интеграции. 
Также он позволяет получить примеры кода на разных языках программирования.


### Формирование запроса

1. На [главной странице](/invest/api) раздела выберите сервис и метод, который нужно протестировать.
2. В блоке **Authorization** введите [токен доступа](/invest/intro/intro/token) и выберите окружение — биржа или [песочница](/invest/intro/developer/sandbox/).
3. В блоке **Body** корректно введите параметры запроса.
4. Нажмите **Отправить запрос**.

<img src="/img/invest/Invest_API.png" width="350"/>


### Обработка ответа

После отправки запроса в блоке **Ответ** отобразится ответ сервера:

- Успешный ответ будет иметь статус `200`. Данные в успешном ответе можно раскрыть, нажав на **+**. Часто структура ответа имеет большую вложенность.
- В случае ошибки в блоке **Ответ** отобразится код ошибки и описание.
  
   <img src="/img/invest/Invest_API_error.png" width="600"/>


#### Заголовки

В ответе сервера также есть заголовки. Они содержат информацию о трекинге запроса, статусе ответа, лимитах и другие данные. Для просмотра заголовков в блоке **Ответ** нажмите на вкладку **Headers**.

<img src="/img/invest/Invest_API_headers.png" width="350"/>

#### Примеры кода

В блоке **Пример запроса** отображаются примеры кода на разных языках программирования. Выберите нужную вкладку и скопируйте код.

<img src="/img/invest/Invest_API_samples.png" width="400"/>


## Swagger

[**Swagger**](https://swagger.io/) — это фреймворк для спецификации RESTful API. Один из его инструментов, Swagger UI, позволяет не только интерактивно просматривать спецификацию, но и отправлять запросы.

**[Swagger UI](https://swagger.io/tools/swagger-ui/)** — один из самых популярных инструментов для создания интерактивной документации. Swagger UI создает интерактивную консоль API для экспериментов с запросами в реальном времени.

### Авторизация

Перед отправкой запросов авторизуйтесь:

1. Нажмите **Authorize**.

    <img src="/img/invest/Auth_swagger1.png" width="650"/>

2. Введите данные в окне **Available authorizations** и нажмите **Authorize**. В нашем примере нужно ввести токен токен доступа. Пример токена — <nobr>`Bearer t.QtEo8ahkNFX4RTpbqp0u4z4GDZq27HzUp6AotJASBx7_DVqmqZMHfM2Cy7JmUjS80boI9eVg`</nobr>.

    <img src="/img/invest/Auth_swagger.png" width="650"/>

### Отправка запроса

Чтобы создать запрос:

1. Разверните конечную точку и нажмите **Try it out**.

    <img src="/img/invest/Swagger_try.png" width="650"/>

   Значение примера в поле **Request body** станет редактируемым.

2. В редактируемом поле измените значения параметров и нажмите **Execute**.

    <img src="/img/invest/Swagger_try2.png" width="650"/>

   Пользовательский интерфейс Swagger отправляет запрос и показывает отправленный curl. В разделе **Responses** можно посмотреть тело и заголовки ответа.

   Если сервер обработал запрос корректно, вернется 200 ответ. Он показывает, как должна выглядеть структура ответа.

   <img src="/img/invest/response_swagger.png" width="650"/>

   <img src="/img/invest/response_swagger2.png" width="650"/>

:::info
Для REST API характерны отдельные [коды ошибок](/invest/intro/developer/error-codes/http_errors).
